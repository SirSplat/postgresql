# Memory Index — `sirsplat/postgresql` project

Retrospective notes captured via `/improve`. Each `feedback_*.md` file documents one accepted finding from a past retrospective.

| File | Topic |
|------|-------|
| `feedback_plan_first.md` | Plan first, await explicit approval before writing/committing/pushing. |
| `feedback_verify_current_state.md` | Probe current state with concrete commands before planning changes. |
| `feedback_audit_docs_adversarially.md` | When alignment-checking docs vs code, tabulate every claim before concluding. |
| `feedback_no_jargon_user_instructions.md` | Define uncommon terms inline; prefer goal-phrased steps over UI clicks. |

These rules are also encoded in the project's `CLAUDE.md` under "Rules for Claude" — the memory files retain the full **why** + **how to apply** rationale that the slim CLAUDE.md rules don't repeat.
